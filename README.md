## شرح المصطلحات (لغير المختصين)

هذا الجدول يوضح معنى المصطلحات التقنية التي ترد في الوثيقة. يمكن الرجوع إليه عند القراءة.

| المصطلح | المعنى المبسط |
|---------|---------------|
| **قاعدة البيانات** | مكان تخزين منظم لبيانات المستخدمين والطلبات (مثل جدول إكسل ذكي). |
| **SQLite** | نوع بسيط من قواعد البيانات، يعمل كملف واحد ولا يحتاج خادماً منفصلاً. |
| **المطابقة الذكية** | آلية تقارن تخصص الطالب ومهاراته مع الجهة وتحدد مدى ملاءمتها (نسبة مئوية). |
| **التشابه الدلالي** | قرب المعنى بين نصين؛ مثلاً "هندسة البرمجيات" و"تطوير البرمجيات" متقاربان في المعنى. |
| **النموذج** | برنامج ذكاء اصطناعي مُدرَّب لفهم النصوص العربية وحساب التشابه بينها. |
| **embeddings** | تمثيل النص كأرقام (متجه) يفهمه الحاسوب؛ النصوص المتشابهة تكون أرقامها قريبة. |
| **Cosine similarity** | مقياس رياضيّ لمدى تقارب نصين؛ كلما زادت القيمة (حتى 1) زاد التشابه. |
| **عتبة** | حد أدنى؛ الجهات التي تقل ملاءمتها عن 40% لا تظهر للطالب. |
| **التدريب (Fine-tuning)** | تحسين النموذج بتمريره على بيانات عربية إضافية لزيادة دقته. |
| **API / مفاتيح API** | واجهة خارجية قد تتطلب اشتراكاً أو رسوماً؛ المنصة لا تعتمد عليها. |
| **Hugging Face** | منصة مجانية تحتوي على نماذج وبيانات جاهزة للذكاء الاصطناعي. |
| **arabic-sts** | مجموعة بيانات عربية تحتوي آلاف أزواج الجمل مع درجات تشابهها (0–5). |
| **GPU** | معالج رسوميات يساعد في تسريع التدريب (اختياري). |
| **fallback** | خطة بديلة؛ إذا فشلت المطابقة الذكية، تُستخدم قواعد ثابتة (كلمات مفتاحية). |
| **التحقق / الاختبار** | في التدريب: بيانات تُستخدم للتأكد من أن النموذج يعمل جيداً قبل الاستخدام الفعلي. |
| **loss (الخسارة)** | مقياس خطأ النموذج؛ كلما انخفضت، زادت دقة النموذج. |

## 1. شرح فكرة المشروع

منصة تعاون هي نظام ويب جامعي يربط بين ثلاثة أطراف رئيسية في عملية التدريب التعاوني: الطلاب الجامعيين، والشركات والجهات الحكومية المستضيفة للتدريب، والمشرفين الأكاديميين. تهدف المنصة إلى تنظيم عملية التنسيق بين هذه الأطراف وتسهيل إدارة طلبات التدريب ومتابعة حالاتها.

## 2. المشكلة التي يحلها المشروع

- **تشتت التنسيق**: عدم وجود قناة موحدة للتواصل بين الطلاب والجهات والمشرفين.
- **صعوبة المتابعة**: صعوبة تتبع حالة طلبات التدريب لكل طالب.
- **إدارة الملفات**: حاجة الطلاب لرفع السير الذاتية وتخزينها بشكل منظم.
- **الفصل بين الأدوار**: كل طرف يحتاج لوحة تحكم خاصة بمهامه (الطالب، الجهة، المشرف).
- **عدم ملاءمة التقديمات**: تقديم الطلاب لجهات غير مناسبة لتخصصاتهم.

## 3. الفئة المستهدفة

- **الطلاب**: طلاب الجامعة المسجلون في برنامج التدريب التعاوني.
- **الجهات**: المؤسسات الحكومية (التابعة للوزارات) والشركات الخاصة التي تستقبل متدربين.
- **المشرفون الأكاديميون**: أعضاء هيئة التدريس المشرفون على برامج التدريب.

## 4. التقنيات المستخدمة

| الطبقة | التقنية |
|--------|---------|
| الواجهة الأمامية | HTML, CSS, JavaScript (أدوات بناء صفحات الويب) |

## 4.1 دقة نموذج المطابقة

| المقياس | القيمة التقريبية |
|---------|------------------|
| نموذج أساسي (قبل التدريب) | تشابه دلالي جيد على 50+ لغة منها العربية |
| بعد التدريب على arabic-sts | تحسن ملحوظ في دقة التشابه الدلالي للنصوص العربية |
| عتبة عرض الجهات | 40% فأعلى (الجهات أقل من 40 لا تظهر للطالب) |
| طريقة الحساب | قياس التشابه (cosine similarity) بين تمثيل النصوص (embeddings)، ثم تحويل النتيجة إلى نسبة 0–100 |

النموذج يدمج بين **المطابقة الدلالية** (تحويل النص لأرقام ومقارنتها) و**القواعد** (ربط التخصصات بكلمات مفتاحية للجهات). عند فشل المطابقة الذكية، تُستخدم القواعد كبديل (fallback).

## 4.2 أمثلة على المطابقة

### المدخلات

| الطالب | الجهة | النتيجة المتوقعة |
|--------|-------|------------------|
| تخصص: هندسة البرمجيات، مهارات: Python, SQL | وزارة الاتصالات، قسم تقنية المعلومات | نسبة عالية (70–95) |
| تخصص: محاسبة، مهارات: Excel | وزارة المالية، الشؤون المالية | نسبة عالية |
| تخصص: هندسة البرمجيات | وزارة الصحة، الموارد البشرية | نسبة منخفضة أو غير ظاهرة |
| تخصص: طب | وزارة الصحة، المستشفيات | نسبة عالية |

```

الجملة 2: التخصص في تطوير البرمجيات والحاسوب
درجة التشابه: 4.2 / 5
```

**التسجيل**: تشفير كلمة المرور (تحويلها لنص غير قابل للقراءة) باستخدام آلية آمنة قبل حفظها.

**تسجيل الدخول**: التحقق من البريد وكلمة المرور، ثم إنشاء جلسة (تذكر أن المستخدم مسجّل الدخول) تحتوي على رقم المستخدم ودوره.

```python
if user and check_password_hash(user['password'], password):
    session['user_id'] = user['id']
    session['role'] = user['role']
    return redirect(url_for('dashboard'))
```

**حماية الصفحات**: آلية تتحقق من أن المستخدم مسجّل الدخول ونوع حسابه (طالب/جهة/مشرف) قبل السماح بفتح الصفحة.

```python
def login_required(role=None):
    def decorator(f):
        @wraps(f)
        def decorated_function(*args, **kwargs):
            if 'user_id' not in session:
                flash('يجب تسجيل الدخول أولاً', 'error')
                return redirect(url_for('login'))
            if role and session.get('role') != role:
                flash('غير مصرح لك بالوصول لهذه الصفحة', 'error')
                return redirect(url_for('dashboard'))
            return f(*args, **kwargs)
        return decorated_function
    return decorator
```

### 7.4 مطابقة الطالب مع الجهات المناسبة

الدالة `match_student_to_companies_ai` تستخدم أداة **sentence-transformers** (نموذج ذكاء اصطناعي متعدد اللغات) لتحويل نصوص الطالب والجهة إلى تمثيل رقمي (embeddings) وحساب التشابه بينها. يعمل كاملاً على جهازك دون الحاجة لاشتراكات أو مفاتيح خارجية.

```python
# ai_matching.py - embeddings دلالية
model = SentenceTransformer('paraphrase-multilingual-MiniLM-L12-v2')
student_emb = model.encode([student_text])
comp_embs = model.encode(comp_texts)
sim = cosine_similarity(student_emb, comp_emb)  # تشابه دلالي
score = (sim + 1) * 50  # تحويل لـ 0-100
```

### 7.6 تحميل السيرة الذاتية - الصلاحيات

السيرة تصل للجهة فعلياً. لا يمكن تحميلها إلا من:
- **الطالب**: سيرته فقط.
- **الجهة**: سيرة طالب تقدّم لها فقط (يوجد طلب في applications).

```python
def download_cv(filename):
    if session.get('role') == 'student':
        if student['cv_file'] == filename:
            return send_from_directory(app.config['UPLOAD_FOLDER'], filename, as_attachment=True)
    elif session.get('role') == 'company':
        app_row = conn.execute(
            'SELECT a.id FROM applications a JOIN students s ON a.student_id = s.id '
            'WHERE s.cv_file = ? AND a.company_id = ?',
            (filename, company['id'])
        ).fetchone()
        if app_row:
            return send_from_directory(app.config['UPLOAD_FOLDER'], filename, as_attachment=True)
    flash('غير مصرح بتحميل هذا الملف', 'error')
    return redirect(url_for('dashboard'))
```

## 10. تدفق النظام

1. **الزائر** يفتح الصفحة الرئيسية ويمكنه تسجيل الدخول أو إنشاء حساب.
2. **إنشاء الحساب**: اختيار نوع الحساب (طالب/شركة-جهة/مشرف) ثم تعبئة النموذج. الجهة الحكومية تختار الوزارة.
3. **تسجيل الدخول**: إدخال البريد وكلمة المرور، ثم التوجيه للوحة التحكم حسب الدور.
4. **الطالب**: يعرض بياناته، السيرة، المهارات، قائمة الجهات المناسبة فقط (مطابقة ذكية)، وحالة طلباته. يجب رفع السيرة قبل التقديم.
5. **الجهة**: تعرض المتقدمين (ذكور/إناث)، المقبولين، المرفوضين، مع إمكانية تحميل السيرة والقبول والرفض والتواصل.
6. **المشرف**: يعرض الجهات (مع الوزارة)، المتدربين، وطلبات التدريب بحالاتها.

## 11. تحسين النموذج (التدريب الإضافي) باستخدام Hugging Face

النموذج الحالي مُدرَّب مسبقاً على لغات متعددة منها العربية. يمكن تحسين دقة المطابقة للنصوص العربية بتدريبه على مجموعة **arabic-sts** (آلاف أزواج الجمل العربية مع درجات تشابهها) من منصة Hugging Face المجانية.

### 11.1 تشغيل سكربت التدريب

```bash
pip install datasets sentence-transformers torch
python train_model.py
```

السكربت يحمّل مجموعة `MohamedRashad/arabic-sts` (أكثر من 11 ألف زوج جمل عربية مع درجات تشابه)، يدرّب النموذج، ويحفظه في مجلد `tawun-match-model`.

### 11.2 استخدام النموذج المُدرَّب

`ai_matching.py` يتحقق تلقائياً: إن وُجد مجلد `tawun-match-model` بجانب الملف، يستخدمه. وإلا يستخدم النموذج الأساسي.

### 11.3 تفاصيل مجموعة arabic-sts

| البند | القيمة |
|-------|--------|
| المصدر | MohamedRashad/arabic-sts على Hugging Face |
| عدد أزواج التدريب | ~11,600 زوج (جملة أولى + جملة ثانية + درجة التشابه) |
| التحقق | ~2,970 زوج (للتحقق من جودة التدريب أثناء العملية) |
| الاختبار | ~2,100 زوج (لتقييم النموذج النهائي) |
| شكل كل زوج | جملة 1، جملة 2، درجة التشابه من 0 إلى 5 |
| المجال | نصوص عربية متنوعة (أخبار، ثقافة، علوم، إلخ) |

### 11.4 المتطلبات

| المورد | الوصف |
|--------|--------|
| بيانات | arabic-sts من Hugging Face (تُحمّل تلقائياً عند التشغيل) |
| GPU | اختياري (معالج رسوميات يسرّع التدريب إن وُجد) |
| وقت | نحو 4 ساعات بدون GPU؛ أقل بكثير مع GPU |
| مكتبات | datasets, sentence-transformers, torch (تُثبّت بأمر واحد) |

### 11.5 نتائج تجربة التدريب الفعلية

تم تشغيل `train_model.py` على جهاز بدون GPU. النتائج التالية توضّح ما يمكن توقعه:
